## 🎯 二、功能完善目标

### 2.1 核心目标
1. **完善审批流程闭环**：从申请发起 → 多级审批 → 审批完成/拒绝 → 结果反馈
2. **增强流程配置能力**：支持流程定义的新增、编辑、版本管理
3. **丰富任务处理功能**：支持转交、委托、加签、会签等高级审批功能
4. **完善审批记录追溯**：完整的审批日志和审计追踪
5. **增加审批统计分析**：审批效率、通过率、超时统计等

## 🛠️ 四、数据库改造计划

### 4.1 现有表结构
```
workflow_definition    - 流程定义表
workflow_node          - 流程节点表
```

### 4.2 需要新增的表
```sql
-- 1. 审批规则表
CREATE TABLE workflow_rule (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    rule_name VARCHAR(100) NOT NULL COMMENT '规则名称',
    rule_type VARCHAR(50) NOT NULL COMMENT '规则类型：AMOUNT/CONDITION',
    condition_expr TEXT COMMENT '条件表达式',
    target_process_key VARCHAR(50) COMMENT '目标流程标识',
    priority INT DEFAULT 0 COMMENT '优先级',
    status VARCHAR(20) DEFAULT 'ACTIVE',
    created_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

-- 2. 审批记录表（详细审计日志）
CREATE TABLE approval_log (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    instance_id BIGINT NOT NULL COMMENT '流程实例ID',
    task_id BIGINT COMMENT '任务ID',
    operator_id BIGINT NOT NULL COMMENT '操作人ID',
    operator_name VARCHAR(50) NOT NULL COMMENT '操作人姓名',
    action VARCHAR(20) NOT NULL COMMENT '操作类型：APPROVE/REJECT/TRANSFER/DELEGATE/ADD_SIGN',
    opinion TEXT COMMENT '审批意见',
    attachment_urls TEXT COMMENT '附件URL列表，JSON格式',
    from_user_id BIGINT COMMENT '转交/委托来源用户ID',
    to_user_id BIGINT COMMENT '转交/委托目标用户ID',
    operation_time DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '操作时间',
    duration_seconds INT COMMENT '处理耗时（秒）',
    ip_address VARCHAR(50) COMMENT '操作IP地址'
);

-- 3. 流程实例扩展表
CREATE TABLE workflow_instance_ext (
    instance_id BIGINT PRIMARY KEY COMMENT '流程实例ID',
    suspended TINYINT(1) DEFAULT 0 COMMENT '是否挂起',
    suspended_reason VARCHAR(200) COMMENT '挂起原因',
    suspended_time DATETIME COMMENT '挂起时间',
    resume_time DATETIME COMMENT '恢复时间',
    urgency_level VARCHAR(20) DEFAULT 'NORMAL' COMMENT '紧急程度：NORMAL/URGENT/EMERGENCY',
    expected_finish_time DATETIME COMMENT '预计完成时间',
    actual_finish_time DATETIME COMMENT '实际完成时间',
    reminder_count INT DEFAULT 0 COMMENT '催办次数',
    last_reminder_time DATETIME COMMENT '最后催办时间'
);

-- 4. 消息通知表
CREATE TABLE workflow_notification (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    user_id BIGINT NOT NULL COMMENT '接收用户ID',
    notification_type VARCHAR(50) NOT NULL COMMENT '通知类型：TODO/RESULT/REMINDER',
    title VARCHAR(200) NOT NULL COMMENT '通知标题',
    content TEXT COMMENT '通知内容',
    instance_id BIGINT COMMENT '关联流程实例ID',
    task_id BIGINT COMMENT '关联任务ID',
    is_read TINYINT(1) DEFAULT 0 COMMENT '是否已读',
    read_time DATETIME COMMENT '阅读时间',
    created_time DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 5. 流程模板表
CREATE TABLE workflow_template (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    template_name VARCHAR(100) NOT NULL COMMENT '模板名称',
    template_code VARCHAR(50) NOT NULL UNIQUE COMMENT '模板编码',
    description TEXT COMMENT '模板描述',
    process_key VARCHAR(50) NOT NULL COMMENT '流程标识',
    nodes_config JSON COMMENT '节点配置JSON',
    is_active TINYINT(1) DEFAULT 1 COMMENT '是否启用',
    usage_count INT DEFAULT 0 COMMENT '使用次数',
    created_by BIGINT COMMENT '创建人ID',
    created_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

## ⚡ 七、实施建议

### 7.1 分阶段实施策略
```
Phase 1 (1周) → 基础功能补全
  ↓ 验收通过后
Phase 2 (1.5周) → 高级功能增强
  ↓ 验收通过后
Phase 3 (1周) → 智能化功能
  ↓
系统上线
```

### 7.2 技术要点
1. **事务管理**：流程实例状态变更必须使用事务，确保数据一致性
2. **并发控制**：使用乐观锁防止任务被重复处理
3. **权限控制**：基于角色的访问控制（RBAC），确保审批权限正确
4. **审计追踪**：所有审批操作记录完整日志，满足内控合规要求
5. **性能优化**：流程历史数据归档，避免单表数据过大

### 7.3 测试重点
- 流程启动和节点流转逻辑
- 多级审批顺序和条件分支
- 转交、委托、加签等特殊操作
- 撤销、终止、挂起等异常处理
- 并发场景下的任务处理
- 审批规则引擎的准确性
