# H2 数据库切换指南

## ✅ 切换完成

项目已成功从 MySQL 切换到 H2 内存数据库！

## 🎯 配置详情

### 1. 依赖配置

**pom.xml** 已添加 H2 依赖：
```xml
<dependency>
    <groupId>com.h2database</groupId>
    <artifactId>h2</artifactId>
    <scope>runtime</scope>
</dependency>
```

### 2. 数据库连接配置

**application.yml** 配置：
```yaml
spring:
  datasource:
    url: jdbc:h2:mem:testdb;DB_CLOSE_DELAY=-1;DB_CLOSE_ON_EXIT=FALSE;MODE=MySQL
    username: sa
    password: 
    driver-class-name: org.h2.Driver
```

### 3. H2 控制台配置

可以通过浏览器访问 H2 控制台查看数据库：
- **URL**: http://localhost:8080/h2-console
- **JDBC URL**: `jdbc:h2:mem:testdb`
- **用户名**: `sa`
- **密码**: （留空）

## ⚠️ 重要提示

### H2 数据库特点：

**优点：**
- ✅ 无需安装 MySQL
- ✅ 启动快速
- ✅ 内存运行，性能优秀
- ✅ 适合开发和测试

**注意事项：**
- ⚠️ **数据存储在内存中** - 应用重启后数据会丢失
- ⚠️ **不适合生产环境** - 仅用于开发测试
- ⚠️ **每次启动都会重新初始化** - 表结构和数据会重新创建

### 如何持久化数据（可选）：

如果需要持久化数据，修改配置文件：

```yaml
spring:
  datasource:
    url: jdbc:h2:file:./data/testdb;MODE=MySQL
```

这样数据会保存在 `./data/testdb.mv.db` 文件中。

## 🔄 切换回 MySQL

如果需要切换回 MySQL：

1. **修改 pom.xml** - 移除或注释 H2 依赖
2. **修改 application.yml** - 恢复 MySQL 配置：
   ```yaml
   datasource:
     url: jdbc:mysql://localhost:3306/test
     username: root
     password: 123456
     driver-class-name: com.mysql.cj.jdbc.Driver
   ```
3. **删除或重命名** schema.sql 和 data.sql
