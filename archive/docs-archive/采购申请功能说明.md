# 采购申请功能说明

## 📋 功能概述

新增了完整的**采购申请流程**，实现了从"发起申请"到"审批处理"的完整闭环。

---

## ✨ 新增功能

### 1. 新建采购申请页面
**路径**: `/workflow/purchase-apply`  
**文件**: `frontend/src/views/workflow/PurchaseApply.vue`

#### 主要功能：
- ✅ **表单填写**：支持填写完整的采购申请信息
  - 基本信息：计划名称、申请部门、需求日期、预算科目
  - 物料信息：物料名称、编码、数量、单价（自动计算总金额）
  - 供应商信息：供应商名称、编码
  - 备注说明：申请事由
  
- ✅ **智能计算**：输入数量和单价后自动计算总金额

- ✅ **双重提交**：
  - **保存草稿**：保存为DRAFT状态，后续可继续编辑
  - **提交审批**：直接提交进入审批流程（PENDING状态）

- ✅ **表单验证**：所有必填项都有严格的验证规则

- ✅ **流程提示**：底部显示审批流程步骤图（4步流程）

---

### 2. 我的申请列表页面
**路径**: `/workflow/my-applications`  
**文件**: `frontend/src/views/workflow/MyApplications.vue`

#### 主要功能：
- ✅ **申请列表**：显示当前用户提交的所有采购申请

- ✅ **状态筛选**：支持按状态筛选
  - 草稿（DRAFT）
  - 待审批（PENDING）
  - 已批准（APPROVED）
  - 已驳回（REJECTED）
  - 采购中（PURCHASING）
  - 已完成（COMPLETED）

- ✅ **搜索功能**：支持按计划名称模糊搜索

- ✅ **操作按钮**：
  - **草稿状态**：提交审批、编辑、删除、查看详情
  - **其他状态**：仅查看详情
  - **已驳回**：可删除重新提交

- ✅ **详情弹窗**：点击"查看详情"显示完整的申请信息

- ✅ **分页支持**：支持10/20/50/100条每页

## 🔄 业务流程

```
用户发起申请
    ↓
填写采购申请表单
    ↓
选择【保存草稿】或【提交审批】
    ↓
├─ 保存草稿 → 状态=DRAFT → 可在"我的申请"中继续编辑
└─ 提交审批 → 状态=PENDING → 进入工作流引擎
    ↓
部门经理审批（DEPT_APPROVE）
    ↓
财务审批（FINANCE_APPROVE）
    ↓
审批完成 → 状态=APPROVED → 生成采购订单
```

## 📊 数据流转

### 前端 → 后端 API

1. **创建采购计划**
   ```
   POST /api/purchase-plan
   Body: { planName, deptCode, materialName, quantity, unitPrice, ... }
   ```

2. **提交审批**
   ```
   PUT /api/purchase-plan/{id}/submit?userId=xxx&userName=xxx
   ```

3. **查询我的申请**
   ```
   GET /api/purchase-plan/page?pageNum=1&pageSize=10
   （前端过滤当前用户的记录）
   ```

4. **删除申请**
   ```
   DELETE /api/purchase-plan/{id}
   ```

## 🚀 使用示例

### 场景1：新建采购申请
1. 点击左侧菜单"审批流程" → "我的申请"
2. 点击右上角"新建申请"按钮
3. 填写表单信息
4. 点击"提交审批"
5. 系统自动跳转到"我的申请"列表

### 场景2：查看申请状态
1. 进入"我的申请"页面
2. 使用状态筛选查看不同状态的申请
3. 点击"查看详情"查看完整信息

### 场景3：处理草稿
1. 在"我的申请"中找到状态为"草稿"的记录
2. 点击"提交审批"按钮
3. 确认后进入审批流程
